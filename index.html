Q1. What is box model?
Answer: A box that wraps around every single html element, includes margin, border, padding, and content. By default width and height are for the content, which is equivalent to content-box.
By setting box-sizing to border-box, it'll make the width and height fit to include content, padding, and border. The two boxes below are example of non-border box and border-box.
Usually we include box-sizing:border-box in the * category in CSS.

Q2. What is specificity?
Answer: Specificity is a weight given to a style to determine if it should be applied or not. For example, if you give a div two different background color in the CSS like below, it'll use the whichever comes last.
div {
background-color: black;
}
div {
background-color: red;
}

If you give div a background-color and the class of the div a background-color, it'll use the class' background-color because it has a higher specificity.
id > class > div/span/etc. If you add !important, then it'll take highest priority, exmaple below.
div {
background-color: red !important
}
Q3. How to center a div?
Answer: First option is to use display: flex, justify-content: center, and align-items: center.
The second option is to use position: relative and position: absolute, then adjust the top, left, and transform to center the inner div.
top: 50%;
left: 50%;
transform: translate(-50%, -50%);
By using top and left, the top left point of the inner div will be at the center of the outer div, then we transform: translate it to center the center point of the inner div to the outer div.
A third option is to use display: flex in the outer div and use margin: auto on the inner div.
display: grid and display: flex have some similarities, so remember to also check that as an option as well.

Q4. What is the difference between "div" and "section"?
Answer: Divs are just used to group elements together, doesn't have a specific meaning.
Sections are used to group together elements that are related to each other, typically structured with headings then paragraph text etc.

Q6: Difference between position relative, absolute, fixed, sticky, and static?
Answer:
Position: static is the default position property. top, right, bottom, left does not affect the positioning of static.
Position: relative is usually used along with position: absolute, it functions similarly to position: static but can be moved by using top, right, bottom, left. It can overlap with other objects on the screen.
Position: absolute makes the object itself no longer associate with other objects on the webpage, and also has a default width of 0, which needs to be changed to whatever you need it to be.
Position: fixed makes the object fix to a position relative to the screenspace, and it'll not move when you scroll. This is useful for making buttons and other functions that needs to be permanently available on the screen.
Position: sticky toggles from position: static to position: fixed. It'll not disappear off the screen. Which is similar to a sticky note.
Q7. Difference between visibility: hidden, and display: none?
visibility: hidden will make an empty render of the space that the object takes, but it is still there and can be interacted. For example, even if you make a button that links to a webpage disappear with visibility: hidden, you can still slick in that space and it'll still do what it's supposed to do.
display: none essentially removes the code that you wrote for that div.
Q8. How to build a triangle in CSS?
Answer: A simple way to make traingle is to use box-sizing: border-box. and then make the content have 0 width and height. This'll make the box get divided to 4 borders, which can make a triangle shape.
There is another way to make more advanced shapes by making a block of object and rotating it.
Q9. What are pseudo elements in CSS?
